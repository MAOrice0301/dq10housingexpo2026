
## 実装方針


カンプファイルのpngがあるのでそれを参考に作成してください。
day-compに関してはday-01からday-05まで同じレイアウトで出展者人数が変わるだけの汎用ページにしてください。

- HTML / CSS / JavaScript のみで作成してください
- JavaScriptライブラリやフレームワークは使わないでください
- 後から自分で編集しやすいように、処理をできるだけ分かりやすく分けてください
- JavaScriptには学習用にコメントを多めに入れてください
- CSSにも、主要なセクションやレイアウトの意図が分かるコメントを入れてください
- GitHub Pagesで公開する予定なので、静的サイトとして動く構成にしてください
- ビルド環境やnpmは使わず、`index.html`を開けば動く形にしてください

私はWebコーディングを勉強中なので、多少コード量が増えても構いません。
短く高度な書き方より、HTML/CSS/JavaScriptの役割が分かりやすい書き方を優先してください。

画像が未完成の箇所は、あとで差し替えやすいように仮画像または分かりやすいプレースホルダーで作ってください。
画像パスは最終配置を想定して記述してください。



## 全ページ共通

・フッターを表示
・共通CSSを利用
・レスポンシブ対応
・スマホでも閲覧可能


## 公開について

- GitHub Pagesで公開予定
- 画像やCSS/JSのパスは相対パスで書いてください
- 外部CDNは使わないでください
- ページ遷移は通常のリンクで行ってください
- ローカルでもGitHub Pagesでも同じように動くようにしてください
